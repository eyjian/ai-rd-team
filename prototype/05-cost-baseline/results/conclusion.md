# P5 结论：成本基线与权重校准

> 状态：**v1 校准完成，后续实际运行中持续校准**

## 核心结论

基于 P1 真实实验（3 成员，~6 分钟，推算 30K tokens）校准 Resource Points 权重：

**新权重**：成员 spawn=40, 消息=2, 广播/接收者=2, 分钟=5, 迭代=15  
**新档位预算**：Lite=120, Standard=400, Full=1500

原权重普遍低估 4 倍，新权重在 P1 数据上偏差 24%（可接受）。

## 对 ai-rd-team 设计的关键影响

| 发现 | 影响 |
|------|------|
| Spawn 成本巨大 | 档位差异以"成员数"为核心维度是对的，精简成员比压缩消息更划算 |
| 广播线性放大 | 严格限制 broadcast 使用（Lite=0/Standard=3/Full=10） |
| 闲置有成本 | 需加 `max_idle_minutes` 超时机制 |

## 局限

- 基于单次实验，置信度低
- 无精确 token API，估算误差 ±30%
- 需要在实际项目中继续验证和校准
